## 1. システム全体像

### 1.1 アーキテクチャ概要
```
[ブラウザ(スマホ/PC)]
   │  HTTPS
   ▼
[Next.js (フロント + APIルート)]
   ├─ 画面描画(SSR/CSR) / 音声I/O(Web Speech API)
   ├─ /api/* (BFF層)
   │     ├─ AIチャット中継 → Anthropic Claude API (ストリーミング)
   │     ├─ レート制限・無料枠カウント
   │     └─ 認証/課金 → CC-Auth Platform SDK
   ▼
[データストア]   [外部サービス]
   ├─ ユーザー/利用回数/進捗  ├─ Claude API
   └─ コンテンツ(CMS or MD)   ├─ (拡張)Amazon Polly / Transcribe
                              └─ CC-Auth (認証・課金)
```

### 1.2 技術スタック（暫定確定）
| レイヤ | 採用 | 備考 |
|--------|------|------|
| フロント | Next.js (App Router) + React + TypeScript | スマホファースト |
| スタイル | Tailwind CSS | 大きな文字・高コントラストのトークン化 |
| 音声出力(TTS) | Web Speech API `speechSynthesis` | 拡張: Amazon Polly |
| 音声入力(STT) | Web Speech API `SpeechRecognition` | 拡張: Amazon Transcribe / Whisper |
| AI | Anthropic Claude API（ストリーミング） | サーバー側で鍵管理 |
| 認証・課金 | CC-Auth Platform SDK | フリーミアム |
| データ | （Phase 2後半で確定：RDB or マネージドDB） | 利用回数・進捗・ユーザー |
| インフラ | AWS（CloudFront + 実行環境） | aws-fast-deploy |

## 3. 画面別仕様（主要画面）

### SCR-01 トップ／オンボーディング（FR-01）
- **目的**: 30秒で「何ができるか」を理解させ、最短で体験へ誘導。
- **要素**:
  - キャッチコピー（大文字・平易）／「今すぐ無料で試す」CTAボタン（最上部・大）
  - 3ステップ説明（①話す/打つ ②AIが答える ③聞ける）をアイコン＋短文で
  - 主要導線カード：体験チャット / チュートリアル / 安全ガイド
- **受け入れ基準**: 初回訪問者がスクロールせずにCTAを認識できる。タップ1回でSCR-02へ。

### SCR-02 AIチャット体験（FR-02, 03, 10, 11）
- **目的**: 登録不要でその場でAIを体験。空欄・タイピングの壁を排除。
- **要素**:
  - チャット表示エリア（吹き出し。AI回答は**ストリーミング**表示）
  - 入力欄 + 送信ボタン（大）
  - **🎤 マイクボタン（FR-11）**: 押下→音声認識→テキスト欄に反映→**送信前に確認/修正**可
  - **お題ボタン群（FR-03）**: 「文章を要約して」等をタップで入力欄に例文投入
  - 各AI回答に **🔊 読み上げボタン（FR-10）**（再生/停止、速度ゆっくり/ふつう）
  - 無料枠残量表示（例：「今日あと 3回」）／上限時はログイン・課金導線
- **状態**: 待機 / 送信中(ストリーミング) / 完了 / エラー / 上限到達
- **受け入れ基準**:
  - 未ログインでも規定回数まで送信できる。
  - マイク非対応/権限拒否時はキーボード入力に**フォールバック**し操作継続可能。
  - 上限到達時は分かりやすいメッセージとログイン/課金導線を表示。

### SCR-04 チュートリアル詳細（FR-04, 05, 10, 11）
- **目的**: 手順に沿って「読む→試す」を1画面で完結。
- **要素**:
  - ステップ表示（1ステップ1アクション、進捗チェック）
  - 各ステップに画面キャプチャ＋平易な説明
  - 本文に **🔊 読み上げボタン（FR-10）**
  - **「この例文で試す」ボタン（FR-05）** → SCR-02の体験チャットへ例文を送る
  - 完了時に達成チェック（FR-12は将来のバッジ拡張）
- **受け入れ基準**: 例文ワンタップで体験チャットに遷移し、その例文が入力済みで開始できる。

### SCR-07 料金プラン／課金（FR-08）
- **目的**: 無料/有料の違いを明示し、迷わず課金完了。
- **要素**: プラン比較表（無料 vs プレミアム）、CTA、CC-Auth課金フロー。
- **受け入れ基準**: 課金成功でユーザーの `plan` が `premium` に更新され、即時に有料機能が解放される。

> SCR-03/05/06/08/09 は同様の粒度で Phase 2 後半に詳細化する（本書では概要のみ）。

## 4. フリーミアム ゲーティング仕様（FR-08）

### 4.1 プランと制限
| 項目 | 無料(free) | 有料(premium) |
|------|-----------|---------------|
| AIチャット回数 | N回/日（既定値: **要確定**） | 大幅増 or 無制限 |
| チュートリアル | 基礎コースのみ | 全コース |
| プロンプト集 | 基本のみ | 全テンプレ |
| 履歴保存 | 不可 | 可 |
| 音声読み上げ/入力 | **利用可（アクセシビリティ）** | 高品質/高精度へ拡張 |

### 4.2 回数カウントの判定ロジック
- **未ログイン**: 端末識別（cookie/localStorage）＋IPで1日上限を管理（厳密でなくUX優先）。
- **ログイン(free)**: ユーザー単位で日次カウント（信頼境界はサーバー）。
- **premium**: カウント上限を緩和/解除。
- カウントは**サーバー側で減算**（クライアント値は信用しない）。日次リセットは利用者ローカル日付ではなくサーバー基準。

## 5. 音声機能 詳細仕様

### 5.1 音声読み上げ TTS（FR-10）
- 既定: `window.speechSynthesis.speak(utterance)`、`lang='ja-JP'`。
- UI: 🔊ボタンでトグル（再生↔停止）。速度は `rate`（ゆっくり=0.8 / ふつう=1.0）。
- 対象: チュートリアル本文、AI回答、主要説明文。
- フォールバック: 非対応ブラウザでは🔊ボタンを非表示（または無効表示）。
- 拡張: 品質要望時、サーバーAPI経由でAmazon Pollyの音声を返す方式へ差し替え可能な抽象化を設ける。

### 5.2 音声入力 STT（FR-11）
- 既定: `webkitSpeechRecognition / SpeechRecognition`、`lang='ja-JP'`、`interimResults=true`。
- フロー: 🎤押下 → 録音/認識 → 認識テキストを入力欄に反映 → **ユーザーが確認・修正** → 送信。
- 権限: 初回にマイク許可をやさしい文言で説明してから要求。
- フォールバック: 非対応/拒否時はキーボード入力に切替（🎤は無効表示）。
- プライバシー: 音声データは文字変換目的のみ。**録音データは変換後に保持しない**（要件5.4準拠）。
- 拡張: 高精度が必要なら音声をサーバーへ送りTranscribe/Whisperで認識する方式に切替可能な抽象化。

## 7. API仕様（BFF / Next.js APIルート、暫定）

| メソッド | パス | 概要 | 認証 | 主な処理 |
|----------|------|------|------|----------|
| POST | `/api/chat` | AIへ送信し回答をストリーム返却 | 任意 | レート判定→Claude中継→カウント減算 |
| GET | `/api/usage` | 残り無料回数取得 | 任意 | 当日カウント参照 |
| GET | `/api/tutorials` | チュートリアル一覧 | 不要 | tier別出し分け |
| GET | `/api/tutorials/:id` | チュートリアル詳細 | 一部要 | premium判定 |
| GET | `/api/prompts` | プロンプト集 | 一部要 | tier別出し分け |
| POST | `/api/auth/*` | 登録/ログイン | — | CC-Auth委譲 |
| POST | `/api/billing/checkout` | 課金開始 | 要 | CC-Auth課金 |
| GET/POST | `/api/me/progress` | 進捗取得/更新 | 要 | FR-12 |
| GET/POST/DELETE | `/api/me/favorites` | お気に入り | 要(premium) | FR-13 |

`POST /api/chat`（例）:
```
Req : { messages: [{role, content}], requestReadAloud?: boolean }
Res : text/event-stream（トークンを逐次送出）
Err : 429(上限到達, {limit, resetAt}) / 401 / 503(AI障害, 再試行可)
```

## 8. 非機能の設計反映

| 区分 | 設計対応 |
|------|----------|
| アクセシビリティ | Tailwindで大文字/高コントラストのデザイントークン化、フォーカス可視、TTS/STT、`aria-*`付与 |
| パフォーマンス | チャットはストリーミング、初期描画はSSR、画像最適化でLCP≤2.5s |
| セキュリティ | API鍵はサーバー保管、入力サニタイズ、プロンプトインジェクション対策、マイクデータ非保持、レート制限 |
| コスト | サーバー側で無料枠減算・上限制御、利用量モニタリング |
| 品質ゲート | TS/ESLintエラー0、テストカバレッジ80%+、セキュリティ80+ |

## 9. 主要ユーザーフロー

**初回・声だけで完結する体験フロー**
```
SCR-01 トップ
  └[今すぐ無料で試す]→ SCR-02 体験チャット
       └ 🎤話す → 認識テキスト確認 → 送信
            └ AI回答(ストリーミング) → 🔊読み上げ
                 └ 無料枠到達 → ログイン/課金導線(SCR-06/07)
```

**学んで試すフロー**
```
SCR-03 一覧 → SCR-04 詳細(🔊で聞く) → [この例文で試す]→ SCR-02 で実行
```
