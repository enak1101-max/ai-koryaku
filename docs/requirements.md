## 1. プロジェクト概要

### 1.1 目的（Vision）
**「30代〜60代の誰もが、迷わず・怖がらず、その場でAIに触れて実践できる」** 入り口となるWebサイトを提供する。

ITやAIに苦手意識を持つ層が「難しそう」「危なそう」という心理的ハードルを越え、**サイト内で実際にAIを体験 → ガイドに沿って自分の課題で実践 → 日常で使い続ける**までを一気通貫で支援する。

### 1.2 解決する課題（Problem）
- AIに興味はあるが「何から始めればいいか分からない」
- ChatGPT等を開いても「何を入力すればいいか分からない」（空欄恐怖）
- 専門用語・横文字が多く、説明を読むだけで離脱する
- 「個人情報は大丈夫？」「お金がかかる？」という不安で踏み出せない

### 1.3 提供価値（Core Value）
> **最重視：とにかく簡単・分かりやすさ**

1. **触ってすぐ分かる** — 登録不要でサイト内のAIをその場で体験できる
2. **迷わない** — ステップ式チュートリアルで「次に何をするか」が常に明確
3. **安心できる** — 安全な使い方・プライバシーの配慮を最初から提示

## 2. ターゲットユーザー（ペルソナ）

| # | ペルソナ | 年代 | 特徴 | ニーズ |
|---|---------|------|------|--------|
| P1 | AI初心者・慎重派 | 50〜60代 | スマホは使うがPC操作に不安。横文字に弱い | まず「触ってみたい」、失敗しても怖くない環境 |
| P2 | 仕事効率化したい実務派 | 30〜40代 | 日常業務で時短したい。忙しい | コピペですぐ使えるテンプレ・具体的な成果 |
| P3 | 学び直し・好奇心派 | 40〜50代 | 新しいことに前向き。継続学習したい | 体系的に学べるステップと達成感 |

### 2.1 デザイン原則（UXガードレール）
- **大きな文字・高コントラスト**（WCAG 2.1 AA 準拠を目標）
- **専門用語ゼロ原則** — どうしても必要な用語には必ずふりがな付き説明
- **1画面1アクション** — 迷わせない。次の一歩を常に1つだけ提示
- **スマホファースト** — ターゲットの多くがスマホ利用を想定
- **読まなくても使える** — 音声読み上げで「目で読む」負担を軽減（シニア配慮）

## 3. スコープ

### 3.1 コア機能（合わせ技：体験 ＋ チュートリアル）
本サイトは以下2つを軸に構成する。

**A. サイト内AIチャット体験**
- サイト上で実際にAI（チャット）をその場で試せる
- 登録不要で数回まで無料体験できる（無料枠）
- 「お題ボタン」をタップするだけで入力例が自動投入され、空欄で悩ませない

**B. ステップ式チュートリアル**
- 「ChatGPTで○○する」等を画面キャプチャ＋手順で解説
- チュートリアル内の例文をワンタップで体験チャットに送って試せる（A と連動）
- 進捗が見える（達成バッジ／チェックマーク）

**C. 音声読み上げ（シニア向けアクセシビリティ）**
- チュートリアル本文・AIの回答・各種説明文を音声で読み上げる
- 各テキスト要素の近くに「🔊 読み上げ」ボタンを配置（ワンタップ再生／停止）
- 再生速度の切り替え（ゆっくり／ふつう）に対応
- 目が疲れる・小さい文字が読みづらいシニア層の「読む負担」を軽減し、ながら聴きにも対応

**D. 音声入力（話しかけてAIに質問）**
- チャット入力欄に「🎤 マイク」ボタンを配置し、話した内容を文字に変換してAIへ送信
- キーボード入力が苦手なシニアでも「話すだけ」で質問できる（空欄恐怖・タイピング負担の解消）
- C（読み上げ）と組み合わせることで、**話す → 聞く** の音声だけで完結する対話体験を提供
- 認識結果は送信前に画面で確認・修正できる（誤認識への安心設計）

### 3.2 フリーミアム構成
| 区分 | 無料 | 有料（プレミアム） |
|------|------|--------------------|
| AIチャット体験 | 1日あたり回数制限あり | 回数大幅増 or 無制限 |
| チュートリアル | 基礎コース | 応用・実務特化コース |
| プロンプト集 | 基本テンプレ | 全テンプレ＋業種別 |
| 保存・履歴 | 制限あり | 履歴保存・お気に入り |
| 音声読み上げ | ◯（アクセシビリティとして全員提供） | 高品質音声・速度調整など拡張 |
| 音声入力 | ◯（アクセシビリティとして全員提供） | 高精度認識・長時間入力など拡張 |

> 音声読み上げはアクセシビリティ配慮のため、基本機能は無料ユーザーにも提供する（有料は高品質音声等の拡張を想定）。

> 課金基盤は CC-Auth Platform SDK（フリーミアム対応）との連携を前提とする。

### 3.3 MVPスコープ（ある程度作り込む）
MVPに**含める**：
- トップ（価値訴求・はじめてガイド）
- サイト内AIチャット体験（無料枠・お題ボタン付き）
- 基礎チュートリアル（5〜10本程度）
- プロンプト集（基本テンプレ）
- 会員登録／ログイン（フリーミアムの無料会員）
- 有料プラン課金（最小構成1プラン）
- 安全に使うためのガイド・FAQ
- 音声読み上げ（チュートリアル本文・AI回答の読み上げ。シニア向けアクセシビリティ）
- 音声入力（マイクで話してAIに質問。シニア向けアクセシビリティ）

MVPに**含めない**（将来）：
- 業種別・上級コンテンツの拡充
- コミュニティ／コメント機能
- 画像生成など他モダリティのAI体験
- 法人向け／チーム機能
- ネイティブアプリ

### 3.4 対象外（Out of Scope）
- AIモデルそのものの自社開発（外部API利用を前提）
- オフライン動作
- 多言語対応（初期は日本語のみ）

## 5. 非機能要件（Non-Functional Requirements）

### 5.1 ユーザビリティ／アクセシビリティ
- WCAG 2.1 AA 準拠を目標（文字サイズ、コントラスト、フォーカス可視）
- 主要操作はタップ3回以内で到達
- 横文字・専門用語には必ず平易な補足
- **音声読み上げ対応** — 主要テキスト（チュートリアル本文・AI回答・説明文）を音声で提供。
  実装は Web Speech API（ブラウザ標準TTS）を基本とし、品質が不足する場合はクラウドTTS（例：Amazon Polly）へ拡張可能とする。スクリーンリーダーとの併用も阻害しないこと。
- **音声入力対応** — マイクで話した内容を文字に変換してAIへ送信できる。
  実装は Web Speech API（SpeechRecognition）を基本とし、精度・対応ブラウザが不足する場合はクラウドSTT（例：Amazon Transcribe / OpenAI Whisper）へ拡張可能とする。
  マイク権限が無い／非対応環境ではキーボード入力にフォールバックし、機能を阻害しない。

### 5.2 パフォーマンス
- 初回表示（LCP）2.5秒以内を目標
- AIチャット応答はストリーミング表示で体感速度を確保

### 5.3 信頼性・可用性
- 稼働率 99.5% 目標
- AI APIエラー時はユーザーに分かりやすいメッセージと再試行導線

### 5.4 セキュリティ（CCAGI品質基準）
- セキュリティスコア 80+ 必須
- 個人情報・会話ログの取り扱いをプライバシーポリシーで明示
- 入力内容のサニタイズ、プロンプトインジェクション対策
- API利用コストの濫用防止（無料枠のレート制限・不正検知）
- **マイク（音声入力）の取り扱い** — 利用時に権限を明示取得し、音声データの送信先・保持方針をプライバシーポリシーで明示。録音データは文字変換後に保持しない方針を基本とする

### 5.5 コスト管理
- AI API利用コストの上限制御（無料ユーザーのレート制限）
- 利用状況のモニタリング・アラート

### 5.6 品質ゲート（CCAGI必須）
- TypeScript エラー 0 / ESLint エラー 0
- テストカバレッジ 80%+

## 8. 前提・制約・リスク

### 8.1 前提
- 日本語ユーザー・国内市場を初期対象とする
- AIは外部API（Claude）を利用する

### 8.2 リスクと対応
| リスク | 影響 | 対応方針 |
|--------|------|----------|
| AI API コスト急増 | 収益悪化 | 無料枠レート制限・コスト監視・上限設定 |
| ターゲットのITリテラシー差 | 離脱 | UXガードレール徹底・ユーザーテスト |
| プロンプトインジェクション等の悪用 | セキュリティ | IronDome等で対策・入力検証 |
| 個人情報の不安 | 登録率低下 | プライバシー明示・安全ガイド先出し |
