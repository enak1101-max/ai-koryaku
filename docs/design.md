---

## 1. デザイン原則（UI版ガードレール）

1. **大きく・はっきり** — 本文16px相当は使わない。基準を**18px以上**に引き上げる。
2. **1画面1アクション** — 主要CTAは画面に1つだけ目立たせる。
3. **専門用語ゼロ** — ボタン文言は動詞で平易に（「送る」「聞く」「話す」）。
4. **押せる場所が分かる** — ボタンは塗り・影・大サイズ。リンクは下線。
5. **声で使える** — 🎤話す / 🔊聞く を常に近くに。
6. **失敗が怖くない** — 取り消し可能・確認あり・やさしいエラー文。

## 2. デザイントークン

### 2.1 カラー（高コントラスト / WCAG AA以上を目標）
| トークン | 値(例) | 用途 | コントラスト |
|----------|--------|------|--------------|
| `--color-bg` | `#FFFFFF` | 背景 | — |
| `--color-text` | `#1A1A1A` | 本文 | 対背景 16:1 |
| `--color-text-sub` | `#444444` | 補足 | 対背景 9:1 |
| `--color-primary` | `#0B6BCB` | 主要CTA(青・信頼感) | 対白文字 4.6:1 |
| `--color-primary-text` | `#FFFFFF` | CTA上の文字 | — |
| `--color-accent` | `#1B8A5A` | 成功・進捗(緑) | AA |
| `--color-warn` | `#B45309` | 注意 | AA |
| `--color-danger` | `#B00020` | エラー | AA |
| `--color-border` | `#CBD5E1` | 枠線 | — |
| `--color-mic-active` | `#B00020` | 録音中マイク | 視認性重視 |

> 色だけに依存しない（状態はアイコン＋テキストでも示す）。

### 2.2 タイポグラフィ（シニア配慮で大きめ基準）
| トークン | サイズ | 行間 | 用途 |
|----------|--------|------|------|
| `--fs-display` | 32–40px | 1.3 | 見出し(トップ) |
| `--fs-h1` | 28px | 1.35 | 画面タイトル |
| `--fs-h2` | 22px | 1.4 | セクション |
| `--fs-body` | **18px** | 1.7 | 本文(基準) |
| `--fs-body-lg` | 20px | 1.7 | 読みやすさ優先本文 |
| `--fs-button` | 18px | 1.2 | ボタン |
| `--fs-caption` | 15px | 1.5 | 補足(最小) |

- フォント: ヒラギノ角ゴ / Noto Sans JP 等の読みやすいゴシック。
- 本文に**ユーザー文字サイズ変更（標準/大/特大）**を用意（FR非機能のアクセシビリティ）。

### 2.3 スペーシング・サイズ
| トークン | 値 | 用途 |
|----------|-----|------|
| `--space-1〜6` | 4 / 8 / 12 / 16 / 24 / 32px | 余白スケール |
| `--radius` | 12px | 角丸(やわらかい印象) |
| `--tap-min` | **48×48px** | タップ領域の最小(指で押しやすい) |
| `--btn-h` | 56px | 主要ボタン高さ |

### 2.4 状態・モーション
- フォーカスリング: 3px `--color-primary`、常時可視。
- アニメーションは控えめ（過度な動きは避ける）。`prefers-reduced-motion`尊重。

## 3. 共通コンポーネント

| コンポーネント | 仕様 |
|----------------|------|
| `PrimaryButton` | 高さ56px・塗り(primary)・18px・角丸12・押下時に明確な反応 |
| `SecondaryButton` | 枠線ボタン。補助操作 |
| `MicButton` 🎤 | 円形・48px以上。録音中は赤＋波形アニメ＋「話してください」表示 |
| `ReadAloudButton` 🔊 | トグル(再生/停止)＋速度切替(ゆっくり/ふつう) |
| `ChatBubble` | user(右・淡色) / ai(左・白＋枠)。ai側に🔊ボタン |
| `TopicChip` | お題ボタン(角丸チップ)。タップで入力欄へ例文投入 |
| `UsageMeter` | 「今日あと N回」を分かりやすく表示。残0で導線 |
| `StepItem` | チュートリアルの1ステップ。チェック・キャプチャ・🔊 |
| `PlanCard` | 料金プラン比較カード |
| `FontSizeToggle` | 標準/大/特大 切替 |

## 4. レイアウト方針
- **スマホファースト**（基準 375–430px）→ タブレット/PCは中央寄せ最大幅 720–960px。
- 下部固定の**ボトムアクションバー**（体験画面：入力欄＋🎤＋送る を常時手元に）。
- ナビは最小限（トップ / 体験 / 学ぶ / マイページ）。アイコン＋日本語ラベル併記。

```

### SCR-02 AIチャット体験
```
┌───────────────────────────┐
│  AIに きいてみよう   今日あと3回 │  ← UsageMeter
│                               │
│  [AI] こんにちは。なんでも…  🔊 │  ← ChatBubble(ai)+読み上げ
│              あなた: 要約して [→]│  ← ChatBubble(user)
│  [AI] わかりました…(入力中▍)  🔊 │  ← ストリーミング
│                               │
│ お題: [要約して][メール下書き][翻訳]│ ← TopicChip

- 残0回: 「今日の無料体験は終わりです」＋[無料会員になる]/[プランを見る]。

## 6. アクセシビリティ実装チェック（WCAG 2.1 AA目標）
- [ ] 本文18px以上、ユーザーで拡大可
- [ ] テキスト/背景コントラスト 4.5:1 以上（大文字3:1）
- [ ] タップ領域48px以上
- [ ] フォーカス可視・キーボード操作可
- [ ] 画像に代替テキスト、アイコンに `aria-label`
- [ ] 状態を色だけに依存しない（アイコン＋文言）
- [ ] スクリーンリーダーとTTSが競合しない
- [ ] `prefers-reduced-motion` 対応

---

## 7. 文言トーン＆マナー（ボイス＆トーン）
- 敬体・短文・前向き。「失敗しても大丈夫」を添える。
- 禁止: 専門用語の素出し（例「プロンプト」→「AIへのお願い文」）。
- ボタン文言例: 「送る」「話す」「聞く」「次へ」「もう一度」「無料で試す」。
- エラー例: 「うまく送れませんでした。もう一度お試しください。」＋[やり直す]。
